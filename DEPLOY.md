# Deploying the RTO Congestion Tool to Google Cloud Run

This guide walks through deploying the app so anyone on your team can access
it via a URL.  Everything is done through a web browser — no software to
install on your computer.

---

## What you'll need

- A Google account with access to your Google Cloud project
  (if your company uses Google Workspace, you likely already have one)
- The `lmp_analysis.duckdb` file (generated by the data loaders)

**Time:** ~25 minutes the first time; ~2 minutes for future data updates.

---

## Part 1 — One-time setup (≈15 minutes)

### Step 1 — Open Google Cloud Console

Go to **console.cloud.google.com** and sign in with your Google account.

In the top bar, click the project selector (it may say "Select a project").
Either pick your existing project or click **New Project** to create one.
Note your **Project ID** — you'll need it in Part 2.

---

### Step 2 — Enable the required APIs

In the search bar at the top, search for and enable each of these APIs.
For each one: search → click the result → click **Enable**.

1. **Cloud Run API**
2. **Cloud Build API**
3. **Artifact Registry API**
4. **Cloud Storage API** *(usually already enabled)*

---

### Step 3 — Create a Cloud Storage bucket

This is where the database file will live.

1. In the left sidebar, go to **Cloud Storage → Buckets**
2. Click **Create**
3. Give the bucket a name, e.g. `yourcompany-lmp-data`
   *(bucket names must be globally unique — add your company name to be safe)*
4. Choose a region — pick **us-central1** or whichever is closest to your team
5. Leave all other settings as default → click **Create**

---

### Step 4 — Upload the database file

1. Click into the bucket you just created
2. Click **Upload files**
3. Select `lmp_analysis.duckdb` from your computer
4. Wait for the upload to finish (the file is ~700 MB — may take a few minutes)

---

## Part 2 — Deploy the app (≈10 minutes)

All of this is done in **Cloud Shell** — a free browser-based terminal that
Google provides.  You don't need to install anything.

### Step 5 — Open Cloud Shell

In the top-right corner of the Cloud Console, click the **>_** (terminal) icon.
A terminal panel will open at the bottom of your browser.

---

### Step 6 — Upload the code to Cloud Shell

In the Cloud Shell toolbar, click the **⋮ (three-dot menu) → Upload** and
upload the following files from your project folder:

- `app_v2.py`
- `Dockerfile`
- `.dockerignore`
- `requirements.txt`

Or, if the code is in a GitHub repository, run:
```
git clone https://github.com/YOUR-ORG/YOUR-REPO.git app
cd app
```

---

### Step 7 — Build and deploy

In Cloud Shell, run the commands below one at a time.
**Replace `YOUR_PROJECT_ID` with your actual project ID** (visible in the top bar of the Console) and **`YOUR_BUCKET_NAME`** with the bucket name from Step 3.

```bash
# Set your project
gcloud config set project YOUR_PROJECT_ID

# Build the container image (takes ~3-5 minutes)
gcloud builds submit --tag gcr.io/YOUR_PROJECT_ID/lmp-app

# Deploy to Cloud Run
gcloud run deploy lmp-app \
  --image gcr.io/YOUR_PROJECT_ID/lmp-app \
  --platform managed \
  --region us-central1 \
  --allow-unauthenticated \
  --memory 2Gi \
  --cpu 1 \
  --timeout 300 \
  --set-env-vars GCS_BUCKET=YOUR_BUCKET_NAME
```

When the deploy finishes, Cloud Shell will print a **Service URL** like:
```
Service URL: https://lmp-app-xxxxxxxxxx-uc.a.run.app
```

That's your app's address. Open it in a browser to confirm it works.
*(The first load takes ~20–30 seconds while it downloads the database — this is normal.)*

---

## Part 3 — Share with your team

Send the Service URL to anyone on your team.
No login is required — the URL is a long random string that isn't publicly
discoverable, so it's safe for internal use.

> **Tip:** Bookmark it in a shared Chrome bookmark folder or pin it in Slack
> so the team can find it easily.

---

## Updating the data

Whenever you run the data loaders and regenerate `lmp_analysis.duckdb`:

1. Go to **Cloud Storage → Buckets → YOUR_BUCKET_NAME**
2. Delete the old `lmp_analysis.duckdb`
3. Upload the new one

The app will automatically use the new file on its next cold start.
To force an immediate refresh, go to **Cloud Run → lmp-app → Edit & Deploy New Revision**
and click **Deploy** without changing anything.  This restarts all instances.

## Estimated costs

At typical internal usage (a few people, a few hours a day):

| Item | Estimated cost |
|------|---------------|
| Cloud Run compute | ~$0–5 / month |
| Cloud Storage (700 MB) | ~$0.02 / month |
| Egress (database downloads) | ~$0.10 / month |
| **Total** | **< $10 / month** |

Cloud Run bills only when the app is actively serving requests and scales
to zero overnight — you won't be charged for idle time.

## Troubleshooting

**App shows a blank page or error on first load**
The database download (~700 MB) can take up to 30 seconds.
Refresh the page after waiting 30 seconds.

**"Permission denied" error in Cloud Shell**
Run `gcloud auth login` and sign in with your Google account.

**Build fails with "API not enabled"**
Go back to Step 2 and make sure all four APIs are enabled.

**Need to redeploy after a code change**
Repeat Step 7 — the build and deploy commands are safe to re-run at any time.
